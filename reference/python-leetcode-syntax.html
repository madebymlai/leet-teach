Python Syntax for LeetCode

The handful of Python mechanics that show up in nearly every LeetCode solution. Keep this open while you code.

Lists

A list is a value, not a function. You cannot call it.

nums = [2, 7, 11, 15]

nums()       # TypeError: 'list' object is not callable
len(nums)    # 4  - the number of elements
nums[0]      # 2  - first element (0-indexed)
nums[-1]     # 15 - last element
nums[i] + nums[j]   # sum of two elements
To get the size of a list, string, or dict, call len(x) - never x().

Looping with indices

for i in range(len(nums)):       # i = 0, 1, 2, ..., len-1
    print(i, nums[i])

for i, val in enumerate(nums):   # index AND value together
    print(i, val)

range(n) yields 0 through n-1. range(a, b) yields a through b-1.

Class vs method scope

Names written directly in the class body are class attributes. They are not visible as local variables inside methods.

class Solution:
    return_list = List[int]      # class attribute - a TYPE, not a value

    def twoSum(self, nums, target):
        return_list              # NameError? No - reads the class attr,
                                 # but List[int] is a type, not a list.
Rule

Declare working variables inside the method, on the first line of the body. The class body is only for typing and structure.

class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        result = []              # declared INSIDE the method
        # ... work ...
        return result

Type annotations vs values

List[int] (after from typing import List) is a type hint - it tells the reader and the type checker what kind of thing a variable is. It is not itself a list. To make an actual empty list, write [] or list().

from typing import List

x: List[int] = []     # annotation says "list of ints"; value is an empty list
y = []                # same thing, no annotation
z = List[int]         # WRONG - z holds the TYPE, not a list

Dictionaries (the hash map)

d = {}                   # empty dict
d[key] = value          # insert / overwrite
val = d[key]            # read - KeyError if missing
val = d.get(key)        # read - None if missing (safer)
val = d.get(key, -1)    # read - -1 if missing

key in d                # True if key exists
for k, v in d.items():  # iterate key/value pairs
A dict is Python's hash map: O(1) average lookup, insert, and delete. This is the workhorse of the complement-pattern.

Returning early

When a problem guarantees exactly one answer, you may return the moment you find it - no need to collect results.

for i in range(len(nums)):
    for j in range(i + 1, len(nums)):
        if nums[i] + nums[j] == target:
            return [i, j]      # found it - done
