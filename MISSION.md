# Mission: LeetCode DSA Practice with AI Coaching

## Why
Walk into a technical-interview coding round and not freeze: read a medium-difficulty problem, recognise which pattern it is, and code a working, complexity-justified solution within the interview clock. The lasting payoff is durable algorithmic fluency that still holds months later, not just an answer for the next interview.

## Success looks like
- Solve an unseen medium independently in under 30 minutes
- Name the pattern of a new problem before writing any code
- State and justify the time and space complexity of any solution on the spot
- Diagnose a failing test case by reasoning about it, not by guessing edits
- Re-derive a previously solved problem after a spacing gap, from understanding rather than memory of the old code

## Constraints
- Terminal-first workflow: helix, leetcode-cli, and the MCP server; no browser IDE
- The AI coaches with progressive hints and syntax discovery; the learner writes every line
- Build pattern understanding, not memorised solutions
- One new pattern at a time, kept inside the zone of proximal development

## Out of scope
- Competitive-programming speed and contest tricks
- System design
- Language trivia unrelated to expressing algorithms
