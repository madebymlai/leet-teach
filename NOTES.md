# Teaching notes

## Preferences
- No em dashes in lesson/reference content. Hyphens are fine.
- Python, not C++, for LeetCode solves (unless a target company requires C++ in interview).
- Likes the `Ctrl-s` save shortcut in helix (bound in `~/.config/helix/config.toml`).
- Prefers pattern lessons to be about the *family*, not a specific problem (Two Sum is the example, not the topic).
- Python syntax lessons live under `lessons/python-syntax/` with their own 2-digit counter; pattern lessons stay in `lessons/` with the 4-digit counter.

## ZPD signals
- Complexity reasoning is partial: states O(1) (per-op) before O(n) (whole pass) when asked. Drill "per operation vs whole solution" explicitly.
- Implementing the pattern outpaces naming it: said "hash lookup" before the canonical "hash-map lookup pattern". Coach the name on each review.
