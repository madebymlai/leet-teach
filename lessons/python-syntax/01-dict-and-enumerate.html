Lesson 01 · Python syntax

dict & enumerate

Two tools you reached for while solving two-sum. This lesson fixes them in memory so next time you write them without checking.

The dict - Python's hash map

A dict is a mutable mapping from keys to values. Average O(1) insert, lookup, and delete. It is the data structure behind the hash-map lookup pattern.

seen = {}          # 1. create - empty dict literal

seen[x] = i        # 2. insert / overwrite - key x, value i

val = seen[x]      # 3. lookup by key - raises KeyError if x not present

x in seen          # 4. membership - True/False, O(1), never raises
The four moves, in the order the pattern uses them

Create once outside the loop. Inside the loop: check membership (complement in seen), then lookup (seen[complement]) only if it is present, then insert (seen[x] = i) after checking. Lookup before insert, always - so an element never matches itself.

The safe lookup: .get

Bare seen[key] raises KeyError if the key is missing. seen.get(key) returns None instead, or a default you choose:

val = seen.get(key)         # None if missing
val = seen.get(key, -1)     # -1 if missing

Use in when you only need the yes/no. Use .get when you need the value and want to handle absence without a branch.

enumerate - index and value in one pass

A built-in that yields (index, value) pairs as you iterate. It replaces the range(len(nums)) + nums[i] dance.

for i, x in enumerate(nums):
    # i = 0, 1, 2, ...
    # x = nums[i]
The bug you just fixed

Writing for x in enumerate(nums) makes x the whole pair (0, nums[0]) - a tuple, not a number. Then target - x fails because you cannot subtract a tuple. Always unpack both names: for i, x in enumerate(nums):.

Optional second argument sets the start index:

for i, x in enumerate(nums, 1):   # i = 1, 2, 3, ...
for i, x in enumerate(nums, 0):   # default

They compose - this is the whole pattern

def twoSum(self, nums, target):
    seen = {}                          # create
    for i, x in enumerate(nums):       # index + value
        complement = target - x
        if complement in seen:         # membership
            return [seen[complement], i]   # lookup
        seen[x] = i                    # insert (after!)
    return []

Every line after seen = {} uses one of the four dict moves or enumerate. There is nothing else to the solve.

You want to know whether key k is in dict d, without crashing if it is not. Which expression?
  • d[k] != None
  • k in d
  • d.contains(k)
  • d.has(k)
What does for x in enumerate(nums) bind x to on the first iteration?
  • The first element, nums[0]
  • The index 0
  • The pair (0, nums[0])
  • The length of nums
In the hash-map pattern, why insert after the membership check, not before?
  • Inserts are faster after reads than before
  • Python dicts forbid insert-then-read in one loop
  • It keeps the dict sorted by insertion order
  • So the current element cannot match itself as its own complement
